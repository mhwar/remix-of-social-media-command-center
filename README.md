# نظام إدارة المحتوى — Content Management Studio

> نظام احترافي لإدارة محتوى العملاء والجهات، مع دليل هوية ذكي قابل للتصدير كسياق جاهز للذكاء الاصطناعي.

## الفكرة

إذا كنت تدير فريقاً يُنتج محتوى لعدة جهات/عملاء، فأنت تعرف المشكلة:
- كل جهة لها ألوانها، خطوطها، نبرة صوتها، ومحظوراتها.
- عند طلب محتوى من أي أداة ذكاء اصطناعي (ChatGPT/Claude/Gemini)، يضيع وقتك في إعادة شرح السياق من الصفر.
- المحتوى الناتج عام، ولا يلتزم بهوية الجهة.

**هذا النظام يحل المشكلة** بجعل كل جهة تُسجَّل مرة واحدة مع دليل هويتها الكامل، ثم بنقرة واحدة تحصل على **System Prompt** جاهز للصقه في أي نموذج ذكاء اصطناعي — ويَنتج محتوى يلتزم بهوية الجهة فوراً.

## الميزات الرئيسية

- 🎨 **ملفات تعريفية كاملة للجهات**: ألوان، شعار، خطوط، قطاع، معلومات تواصل.
- 📘 **دليل جهة (Brand Guide) شامل** لكل عميل:
  - الهوية البصرية (ألوان، خطوط)
  - نبرة الصوت وشخصية العلامة
  - مصطلحات مفضلة/محظورة
  - الجمهور المستهدف والمنصات النشطة
  - السياسة الإعلامية والملاحظات القانونية
  - استراتيجية الهاشتاق وأمثلة معتمدة
- ✨ **مولد سياق الذكاء الاصطناعي** — الميزة المحورية:
  - Prompt جاهز منسّق تلقائياً من كل بيانات الجهة
  - تخصيص حسب المنصة/نوع المحتوى/الطول
  - زر واحد للنسخ + زر لتنزيل Markdown
  - روابط مباشرة لفتح ChatGPT وClaude وGemini
- 📋 **لوحة Kanban** لتتبع المشاريع والمهام (فكرة → كتابة → مراجعة → معتمد → منشور)
- 📅 **تقويم محتوى شهري** ملون بألوان كل جهة
- 🏷️ تصفية المهام حسب الجهة
- 🌐 واجهة عربية كاملة مع دعم RTL

## المكدّس التقني

| الطبقة | التقنية |
|---|---|
| Framework | Next.js 15 (App Router) + TypeScript |
| Styling | Tailwind CSS + RTL native |
| Database | SQLite + Prisma ORM |
| Validation | Zod |
| Icons | Lucide React |

## البدء السريع

```bash
# 1. تثبيت التبعيات
npm install

# 2. تهيئة قاعدة البيانات
npx prisma db push

# 3. تعبئة بيانات تجريبية (جهة + مشاريع + مهام)
npm run db:seed

# 4. تشغيل الخادم
npm run dev
```

افتح [http://localhost:3000](http://localhost:3000).

## كيف يعمل مولد سياق الذكاء الاصطناعي؟

الملف `src/lib/ai-prompt-builder.ts` يحتوي على دالة واحدة `buildSystemPrompt()` تأخذ كائن `Client` مع `BrandGuide` وتُنتج نصاً Markdown منسّقاً يحتوي على:

1. **مقدمة تعريفية** بالجهة واسمها وقطاعها.
2. **نبرة الصوت والشخصية** — موجّهة للنموذج كتعليمات.
3. **المصطلحات** (مفضلة ✓ ومحظورة ✗).
4. **الجمهور المستهدف والمنصات**.
5. **السياسة الإعلامية والملاحظات القانونية**.
6. **استراتيجية الهاشتاق**.
7. **أمثلة محتوى معتمدة** للاسترشاد بالأسلوب.
8. **المهمة المطلوبة** مع تخصيص (نوع المحتوى، المنصة، الطول، الموضوع).
9. **متطلبات نهائية** تذكّر النموذج بالالتزام بكل ما سبق.

### مثال على الإخراج:

```markdown
# سياق العلامة التجارية للمحتوى

أنت الآن تكتب محتوى باسم **شركة نور للتقنية** (Noor Tech).
التزم بكل التعليمات التالية بدقة...

## عن الجهة
- **القطاع:** التقنية والبرمجيات
- **الوصف:** شركة متخصصة في حلول البرمجيات السحابية...

## نبرة الصوت والشخصية
- **النبرة:** احترافي — متوازن بين الجدية والمرونة
- **الشخصية:** نتحدث كخبير تقني ودود...

## المصطلحات
**✓ استخدم هذه المصطلحات:**
- الحلول السحابية
- الأتمتة الذكية

**✗ تجنب هذه المصطلحات تماماً:**
- رخيص
- بسيط جداً

...

## المهمة الآن
اكتب **منشور** لمنصة **تويتر / X** باللغة **العربية** بطول تقريبي **280 حرف**.

**الموضوع/التفاصيل:** إعلان إطلاق المنتج الجديد.

### المتطلبات النهائية:
- التزم التام بنبرة الصوت والشخصية أعلاه.
- تجنب المصطلحات المحظورة...
```

انسخ هذا والصقه في ChatGPT أو Claude كأول رسالة، ثم اطلب المحتوى.

## بيانات تجريبية

بعد تشغيل `npm run db:seed`:
- **جهة:** شركة نور للتقنية (مع دليل هوية مكتمل)
- **مشروع:** حملة إطلاق المنتج الجديد
- **مهام:** 4 مهام في حالات مختلفة
